# Local SEO Four Pages Design

**Topic:** `/services` `/cases` `/recruit` `/contact` を八戸市・青森県のローカルSEO文脈で強化する

**Goal:** 主要4ページを、検索エンジンとユーザーの両方に対して「八戸市・青森県で電気設備工事・施工実績・採用・問い合わせを探している人向けのページ」として明確に伝わる状態にする。

**Scope:**
- 既存4ページの本文・導線・構造化データを強化する
- 新規ページは作らない
- デザイン崩れを避け、現行UIのトーンを維持する

**Non-Goals:**
- サービス別の新規LP作成
- 求人管理機能や求人APIの追加
- Search Console や Google ビジネスプロフィールの運用代行

## Approach Options

### Option A: 本文 + 構造化データを4ページだけ深掘り

各ページの冒頭説明、補足セクション、CTA 周辺文言をローカル検索意図に合わせて調整し、ページ種別に応じた schema を追加する。

**Pros**
- 既存構成を崩さず実装できる
- 早く本番へ出せる
- 検索エンジンへの意味付けが増える

**Cons**
- ページ数自体は増えないため、狙えるクエリは限定的

### Option B: A + 地域特化FAQを各ページに追加

Option A に加え、ページ末尾へ短い地域特化FAQを追加する。

**Pros**
- ロングテール対策として強い
- schema との相性が良い

**Cons**
- 文章量が増え、UI調整の工数が上がる
- 今回は4ページ同時実装だとやや過剰

### Option C: 地域LPを新規作成

`八戸市の電気工事` など専用ページを増やす。

**Pros**
- 中長期では最も強い

**Cons**
- 情報設計と導線設計が別途必要
- 今回の「技術SEOを先に」の範囲を超える

## Recommended Approach

Option A を採用する。理由は、既存の prerender 化と相性が良く、短いサイクルで検索エンジンが理解しやすい情報量を増やせるため。

## Page-Level Design

### `/services`

- ヒーロー説明を「八戸市・青森県で対応する電気設備工事会社」という文脈に寄せる
- サービス一覧の前後に「対応エリア」「相談対象」「設計施工から保守までの体制」を補足する
- `Service` / `OfferCatalog` をページ固有 schema として追加する

### `/cases`

- 施工実績ページ冒頭に、八戸市・青森県を中心とした実績であることを明示する
- 一覧の前に、対象施設や工種の幅を要約する
- `CollectionPage` 相当の説明強化と、実績ページとしての地域文脈を JSON-LD へ反映する

### `/recruit`

- 八戸市周辺で働く電気工事・設備工事の仕事として伝わる文言を追加する
- 地域インフラを支える仕事であること、未経験者歓迎と資格取得支援の文脈を前段で整理する
- 将来的な `JobPosting` に備えつつ、今回はページ本文の検索意図整合を優先する

### `/contact`

- 「八戸市・青森県で電気設備工事を相談したい法人・個人向け窓口」であることを明示する
- 電話、見積、工事相談、採用相談の導線を整理する
- `ContactPage` の意味づけを JSON-LD へ追加する

## Content Rules

- 不自然なキーワード詰め込みは避ける
- `八戸市`, `青森県`, `電気設備工事`, `電気通信工事`, `消防設備`, `太陽光発電`, `施工実績`, `採用`, `問い合わせ` を文脈に合わせて使う
- 既存のブランドトーンは維持する

## Verification

- `npm run build` が通ること
- prerender 後の `dist/services/index.html`, `dist/cases/index.html`, `dist/recruit/index.html`, `dist/contact/index.html` で title/description/canonical が維持されること
- 各ページ本文に地域文脈が追加されていること
- 追加 schema が prerender HTML に含まれること
